# 服务异常处理手册

## 门店相关异常

### 门店闭店

**触发条件**：call_service(pet_store_mock, list_locations) 返回的门店 closed_dates 包含查询日期

**处理方式**：
1. 告知用户门店当天闭店
2. 推荐最近的开门日期
3. 如果有其他门店开门，推荐替代门店

### 门店不在服务范围

**触发条件**：用户位置超出所有门店 service_radius_km

**处理方式**：
1. 告知用户暂无覆盖该区域的服务门店
2. 建议用户到最近的门店自取或自行前往

## 档期相关异常

### 当天无可用档期

**触发条件**：call_service(pet_store_mock, query_availability) 返回空列表

**处理方式**：
1. 告知用户当天已约满
2. 推荐最近日期的可用档期
3. 如果用户急需，建议换门店

### 预约时间不足

**触发条件**：服务时长超过剩余可用连续档期

**处理方式**：
1. 告知用户连续时间不足
2. 推荐可用的分散时间段（如果服务可拆分）
3. 推荐其他日期或其他门店

### 最小提前时间不满足

**触发条件**：预约时间距当前时间不足 min_advance_minutes（60 分钟）

**处理方式**：
1. 告知用户需提前至少 1 小时预约
2. 推荐满足提前时间的最近档期

## 库存相关异常

### 商品库存不足

**触发条件**：call_service(pet_store_mock, query_inventory) 返回库存为 0

**处理方式**：
1. 告知用户该门店商品暂无库存
2. 查询其他门店库存
3. 如果其他门店有库存，建议换门店或跨店配送
4. 如果所有门店无库存，建议替代商品或设置补货提醒

### 商品全部缺货

**触发条件**：所有门店库存均为 0

**处理方式**：
1. 告知用户该商品暂时缺货
2. 推荐同类替代商品
3. 使用 create_notification 设置补货提醒

## 打车/运力相关异常

### 宠物专车不可用

**触发条件**：call_service(pet_transport_mock, estimate_transport/request_transport) 返回无可用车辆

**处理方式**：
1. 告知用户暂无宠物专车
2. 建议用户自行前往门店
3. 保留预约不取消

### 同城配送不可用

**触发条件**：call_service(pet_transport_mock, estimate_transport/request_transport) 返回配送不可用

**处理方式**：
1. 告知用户同城配送暂不可用
2. 建议到店自取
3. 询问是否继续订单

## 日历相关异常

### 日历权限未授权

**触发条件**：query_calendar 返回权限不足错误

**处理方式**：
1. 告知用户需要日历权限来检查冲突和设置提醒
2. 询问是否授权
3. 如果不授权，跳过冲突检查，继续流程

### 日程冲突

**触发条件**：query_calendar 返回的事件列表中存在与预约时间重叠的事件

**处理方式**：
1. 告知用户存在日程冲突
2. 展示冲突事件
3. 推荐无冲突的替代时间段

## 通用异常处理

### 服务调用超时

**触发条件**：call_service 或 use_skill 调用超时

**处理方式**：
1. 告知用户服务暂时繁忙
2. 建议稍后重试
3. 发出 task_error 事件

### 数据不一致

**触发条件**：Mock 数据状态与预期不符（如预约已不存在但仍尝试修改）

**处理方式**：
1. 重新查询最新状态
2. 基于最新状态重新推荐方案
3. 发出 task_error 事件

### 未知错误

**触发条件**：未预期的错误

**处理方式**：
1. 告知用户遇到问题
2. 保存当前流程状态
3. 提供可恢复的下一步建议
4. 发出 task_error 事件

## 取消/修改相关异常

### 取消预约失败

**触发条件**：`call_service(pet_store_mock, cancel_booking)` 返回失败

**处理方式**：
1. 告知用户取消失败
2. 检查预约状态（可能已取消或已完成）
3. 如预约仍存在，建议联系门店人工取消
4. 如预约已不存在，告知用户无需取消

### 修改预约失败

**触发条件**：`call_service(pet_store_mock, modify_booking)` 返回失败

**处理方式**：
1. 告知用户修改失败
2. 检查失败原因（档期已满、预约已取消等）
3. 如档期已满，推荐其他时间段或门店
4. 如预约已取消，建议重新预约

### 取消专车失败

**触发条件**：`call_service(pet_transport_mock, cancel_transport)` 返回失败

**处理方式**：
1. 告知用户专车取消失败
2. 检查专车状态（可能已在执行中或已完成）
3. 如专车已在路上，告知用户司机即将到达
4. 如专车已出发，建议直接告知司机

### 修改专车时间失败

**触发条件**：`call_service(pet_transport_mock, modify_transport)` 返回失败

**处理方式**：
1. 告知用户修改失败
2. 检查失败原因（新时间无可用车辆等）
3. 推荐其他可用时间段
4. 如无可用车辆，建议取消专车自行前往

### 取消订单失败

**触发条件**：`call_service(pet_store_mock, cancel_order)` 返回失败

**处理方式**：
1. 告知用户取消失败
2. 检查订单状态（可能已发货、已完成）
3. 如已发货，建议拒收或联系配送员
4. 如已完成，告知无法取消

### 级联取消异常

**触发条件**：取消预约成功但关联的专车/订单取消失败

**处理方式**：
1. 告知用户部分取消成功
2. 列出成功和失败的项目
3. 对失败项提供人工处理建议
4. 记录异常，后续重试
